Use this template when creating new projects in Linear. Fill in each section concisely to ensure clarity and alignment.

```markdown

**Owner:** [Full Name (Role)]

**Contributors:** [Team Member 1 (Role)], [Team Member 2 (Role)]

## Background & Rationale

[1-3 sentences explaining the problem/opportunity and why this project matters]

**Company Alignment:** [How this project supports company OKRs or strategic goals]

## Goals & Objectives

**Primary Goal:** [Main objective]

**Key Results:**

- [Measurable outcome 1]
- [Measurable outcome 2]
- [Measurable outcome 3]

**Success Metrics:**

- [Metric 1 with target]
- [Metric 2 with target]

## Scope

**In-Scope:**

- [Deliverable/task 1]
- [Deliverable/task 2]

**Out-of-Scope:**

- [Not included 1]
- [Not included 2]

**Dependencies:**

- [Dependency 1]
- [Dependency 2]

## Timeline

**Start Date:** [Date]
**End Date:** [Date]

**Milestones:**

- [Milestone 1] - [Date]
- [Milestone 2] - [Date]
- [Milestone 3] - [Date]

## Risks & Mitigation

**Known Risks:**

- [Risk 1]: [Mitigation strategy]
- [Risk 2]: [Mitigation strategy]

- [Plan 1]
- [Plan 2]

```

## Example (Development Project)

```markdown
# Onboarding Flow Redesign

**Owner:** Jane Doe (Product Designer)

**Contributors:** John (Frontend), Sara (Backend), Leo (UX Research)

## Background & Rationale

New user activation rate is only 20%. We've received user feedback about the sign-up process being confusing.

**Company Alignment:** Supports Q1 company OKR to increase activation from 20% to 40%.

## Goals & Objectives

**Primary Goal:** Redesign and launch a more intuitive user onboarding flow

**Key Results:**

- Launch redesigned user onboarding flow by Mar 31
- Increase new user activation rate from 20% to 40% within one month of launch
- Reduce drop-off during signup by 50%

**Success Metrics:**

- User activation rate (target: 40%)
- Time to complete onboarding (target: under 2 minutes)

## Scope

**In-Scope:**

- Redesign signup pages
- Create first-use tutorial
- Implement frontend and backend changes
- Create a tutorial video
- A/B test new vs. old flow (50% of new users)

**Out-of-Scope:**

- In-app referral program (separate project)
- Mobile app onboarding (future phase)

**Dependencies:**

- Design system components
- Analytics implementation

## Timeline

**Start Date:** Feb 1
**End Date:** Mar 31

**Milestones:**

- Design completion - Feb 15
- Development completion - Mar 25
- Testing completion - Mar 30
- Launch - Mar 31

## Risks & Mitigation

**Known Risks:**

- User feedback delay: Schedule research sessions early
- No dedicated QA engineer: Developers will double up on testing

**Contingency Plans:**

- If timeline slips, prioritize core flow and defer tutorial video

## Status & Updates

**Current Status:** Planning phase (as of Jan 20)

**Latest Update:** Research plan finalized, design kickoff scheduled
```

## Example (Operational Project)

```markdown
# Customer Support Process Implementation

**Owner:** Alex Roe (COO)

**Contributors:** Kim (Support Lead), Dev (Engineer), Mike (Data Analyst)

## Background & Rationale

Support requests are currently handled ad-hoc, leading to slow response times and inconsistent service as we scale.

**Company Alignment:** Supports our Q2 goal to improve customer satisfaction and prepare for growth.

## Goals & Objectives

**Primary Goal:** Implement a standardized customer support workflow

**Key Results:**

- Document and implement support workflow by May 15
- Achieve average first-response time < 4 hours
- Reach customer satisfaction score of 90%+

**Success Metrics:**

- Response time (target: < 4 hours)
- Customer satisfaction score (target: 90%+)
- Ticket resolution rate (target: 85% same day)

## Scope

**In-Scope:**

- Evaluate and select a support ticketing tool
- Import existing tickets into new system
- Define SLA guidelines for responses
- Train team on new process
- Publish customer-facing Support FAQ

**Out-of-Scope:**

- 24/7 support coverage
- Live chat implementation

**Dependencies:**

- IT infrastructure for tool deployment
- Legal review of SLAs

## Timeline

**Start Date:** Mar 1
**End Date:** May 15

**Milestones:**

- Tool selection - Mar 15
- System setup - Apr 1
- Process documentation - Apr 15
- Team training - May 1
- Launch - May 15

## Risks & Mitigation

**Known Risks:**

- Data migration issues: Run test migrations first
- Team adoption challenges: Schedule extra training sessions

**Contingency Plans:**

- Maintain current email support as backup during transition

## Status & Updates

**Current Status:** In Progress (as of Apr 20)

**Latest Update:** Tool selected, data migration underway
```
